工厂里来了AI Agent，但它不是来替代你的

2026年6月12日 · AI · 智能制造 · AI Agent

一、为什么是现在？

制造业的日常决策量远超外界想象。一个中型注塑车间每天产生的决策点超过 2000 个：工艺参数微调、模具切换排程、来料异常判定、设备报警响应、质检抽样策略。这些决策大多数由一线班组长凭经验做出，少数触发工程师介入，极少数上报到管理层。

这个系统的脆弱性在于：经验随人走。一个在车间干了八年的老师傅离职，带走的不是一个人，是一整套隐性知识——哪种模具配哪台机器最稳定、某个产品的"甜区"在什么模温范围、什么声音代表螺杆磨损到了临界点。

制造业真正的痛点不是"缺数据"——MES 系统跑了五年，数据库里躺着几千万条工艺记录。痛点是数据到决策的最后一百米走不通：数据在系统里，经验在人脑里，两者之间没有桥梁。

二、三代演进：从脚本到自主决策

理解 AI Agent 不能跳过它的进化谱系。制造业智能化经历了三个清晰的代际：

第一代：固定规则自动化（1980s-2010s）

PLC 程序、DCS 控制回路、MES 固定流程。特点是 if-then 全覆盖，但系统零弹性——来料批次波动超过预设阈值就报警停机，无法自主判断"这个波动是否真的影响质量"。本质是自动化的，不是智能化的。

第二代：RPA + 规则引擎（2015-2022）

在固定规则上加了一层流程自动化。财务对账、采购审批、工单流转被 RPA 接管。但 RPA 的致命伤是：规则由人编写，它自己不会学。一个采购审批的 RPA 跑了两年，从来没有意识到"金额低于 5000 元且供应商评级 A 的订单其实可以自动通过"——因为它看不到模式，只看到规则。

第三代：AI Agent（2023-至今）

与前两代的本质区别：Agent 具备感知-推理-行动-学习的闭环。它不是被动的规则执行器，而是能够理解上下文、分解任务、调用工具、自主决策的智能体。

一个比喻理清三代差异：
第一代 = 轨道电车（只能在铺设的轨道上跑）
第二代 = 遥控汽车（人可以远程控制但车不会自己认路）
第三代 = 自动驾驶——有地图、有传感器、有决策算法，能在陌生路况下自己决定怎么走

三、AI Agent 的核心架构

一个能在制造业落地的 AI Agent 包含四个核心模块，缺一不可：

1. 感知层（Perception）

不只是数据读取，而是语义理解。当 MES 报出"3 号注塑机保压压力波动"，Agent 需要理解这是工艺参数层面的问题还是传感器本身的问题——两种情况的处理路径完全不同。感知层需要融合三类数据：结构化数据（传感器时序）、半结构化数据（质检报告文本）、非结构化数据（维修工单的手写备注）。

2. 知识层（Knowledge）

这是 Agent 与通用 LLM 最大的区别。通用 LLM 知道"注塑"的定义，但不知道"你们车间 5 号机的止逆阀上周刚换过"。知识层整合了三种知识：行业知识（注塑工艺理论）、企业知识（设备台账、模具履历、历史质量数据库）、岗位知识（操作规范、异常处理 SOP）。知识层的质量直接决定 Agent 的决策质量——垃圾进垃圾出，在 Agent 领域同样适用。

3. 推理与规划层（Reasoning & Planning）

这是 Agent 的"大脑"。当接收到一个任务——比如"3 号机产品质量波动，找出原因"——Agent 需要完成三步：拆解任务（是设备问题还是材料问题还是工艺问题）、制定排查计划（先查哪条数据、再验证哪个假设）、动态调整（如果 A 路径走不通，自动切换到 B 路径）。

4. 执行层（Action）

Agent 的"手脚"。与传统自动化最大的区别是：执行层的工具集可以动态扩展。传统系统能做的只有"停机"或"报警"两种动作。Agent 可以调用更多工具：查询历史维修记录、调取上次换模后的工艺曲线、甚至通过企微或钉钉直接给工程师发消息问"上次这个模具换的是什么材质的止逆环？"

四、三个正在发生的落地场景

场景一：工艺参数自主调优

杭州一家汽配注塑企业部署了 Agent，接管了 12 台注塑机的保压参数微调。逻辑是这样的：每 5 分钟采集一次产品重量数据（通过机械手取件后的在线称重），当连续 3 个样本的重量偏差超过 0.05g，Agent 自动分析偏差方向（偏重还是偏轻）、关联当前模温/料温/环境温度/材料批次，然后给出保压压力或保压时间的微调建议——并在获得工程师确认后自动下发到注塑机控制器。

实施 3 个月后的结果：产品重量 CPK 从 1.18 提升到 1.52，夜间异常响应时间从平均 23 分钟缩短到 4 分钟。更重要的是：以前凌晨两点参数飘移，夜班操作工要么没发现要么不敢调，等到早上工程师上班已经废了半个班的料。现在 Agent 在 30 秒内就能发出预警和调整方案。

场景二：设备异常预判

苏州一家电子制造企业的 SMT 产线 Agent 做的不是事后报警，而是事前预判。通过持续学习贴片机的主轴电流波形、真空压力曲线和抛料率，Agent 能提前 2-4 小时预警供料器即将卡料或吸嘴即将堵塞，准确率约 87%。这不是简单的阈值触发——阈值只能告诉你"已经到了临界点"，Agent 能告诉你"按当前趋势，3 号供料器可能在 2 小时后卡料，建议在下一批次换线时提前清理"。

场景三：跨系统知识问答

这不是 ChatGPT 套个壳。东莞一家包装企业的 Agent 接入了 MES、ERP、WMS 三个系统，一线操作工可以直接用自然语言问："上次做 30ml 乳液瓶的时候模具温度是多少？""这批 PE 料是哪天入库的？IQC 结果怎么样？"——Agent 自动跨系统检索并给出整合答案。以前这些信息分散在三个系统的不同界面里，查一次要 5-15 分钟，现在 30 秒。

五、别踩这三个坑

坑 1：把 Agent 当成万能的

Agent 不是超人。它的决策质量上限 = 知识层质量 × 推理能力，两者都有边界。把 Agent 放到一个知识贫瘠、数据缺失的环境中，它比一个经验丰富的老师傅差得远。Agent 的正确定位是：让好的决策可复制，而不是替代所有决策。

坑 2：跳过知识梳理直接上模型

这是目前制造业 AI 项目失败率最高的原因。企业花 200 万买大模型 API，结果发现它不知道"你们车间 5 号机的锁模力上限是多少"。

Agent 项目时间分配的黄金法则：
60% 在知识梳理和数据治理
30% 在系统集成和工具对接
10% 在模型调试
很多企业完全搞反了——90% 时间花在调模型，10% 糊弄数据，结果就是半年后项目死掉。

坑 3：追求全自动忽略人机协同

制造业不是互联网。一个错误的工艺参数下发可能直接损坏模具（几十万）或产生批量废品。Agent 的合理形态是人在回路（Human-in-the-Loop）——Agent 做分析和建议，关键操作由人确认。随着 Agent 在特定场景的准确率积累，逐渐提高自主决策比例，而不是一上来就追求"全自动"。

六、未来三年

三个趋势值得关注：

第一，从单点 Agent 到多 Agent 协作。未来的车间不会只有一个 Agent，而是设备 Agent、质量 Agent、计划 Agent、物料 Agent 协同工作。多 Agent 之间的信息交换和冲突消解会是下一个技术前沿——比如质量 Agent 建议降速保良率，而计划 Agent 要求提速赶交期，谁来仲裁？

第二，从辅助决策到自主执行。随着 Agent 在特定场景的可靠性被验证，关键操作的控制权会逐步下放。这不是技术问题，是信任积累和风险管理的问题。需要一个渐进式放权框架：观察期（只分析不操作）→ 辅助期（分析+建议+人工确认）→ 自主期（特定场景自动执行）。

第三，Agent 将改变制造业的人才结构。当日常参数调优、异常判定、信息查询被 Agent 接管，工程师的时间将从"救火"转向"防火"——更多精力投入到工艺研究、质量体系优化和新产品导入。这对工程师的要求不是降低了，而是提高了：你需要理解 Agent 的决策逻辑，才能在它出错时快速定位问题。

结语

这本质上不是一次技术升级，而是一场制造业认知模式的迁移：从"人记住了就是经验"到"系统学会了才是能力"。那些率先完成这场迁移的企业，五年后的竞争力差距将大到令人不安。

AI Agent 不是来替代你的。它是来替代那些重复、琐碎、依赖记忆的决策——让你有时间做只有人类才能做的事：理解复杂系统、做出战略判断、创造新的可能。
